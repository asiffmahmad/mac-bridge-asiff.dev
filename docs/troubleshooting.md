# Troubleshooting & Diagnostics

## Common Issues

### 1. Connection Errors / "Bridge Unreachable"
- **Check Tunnel Domain**: Ensure you configured the correct https tunnel URL (e.g. `https://mac.mydomain.com`) inside the frontend console, not a local `localhost` IP.
- **Check Backend Daemon Status**:
  ```bash
  launchctl list | grep com.bridge.macbridge
  ```
- **Inspect Daemon Logs**:
  ```bash
  tail -f ~/.mac-bridge/bridge.log
  ```

### 2. Authentication Failures / "401 Unauthorized"
- **Bcrypt Password Matching**: If you forgot the password, retrieve the autogenerated key:
  ```bash
  cat ~/.mac-bridge/users.json
  ```
- **Reset credentials**: Delete `~/.mac-bridge/users.json` and restart the backend. A new password will be generated and printed to standard output.
- **Token Expiry**: The client should refresh keys automatically. If you get persistent 401s, verify the device clock is synchronized.

### 3. Path Traversal Blocks / "403 Forbidden" on Files
- **Directory Restrictions**: The backend restricts file access to the user home folder. Trying to access `/etc/`, `/var/`, or other root folders will block.
- **Paths Syntax**: Ensure directories start with `~/` or `/Users/username/`. Using bare relative paths or traversal notation (`..`) to escape the home folder is blocked by design.

### 4. Websocket Disconnects
- **Proxy/Tunnel Issues**: Some HTTP proxies and tunnels terminate inactive WebSocket connections. Ensure heartbeats are active (the default is configured to 10 seconds).
- **Check Port Config**: WebSocket uses the main HTTP port (8080). If you have other local servers running on 8080, port conflicts might cause silent disconnects. Check local ports:
  ```bash
  lsof -iTCP:8080 -sTCP:LISTEN
  ```
